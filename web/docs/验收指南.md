## 新增验收点（M3：高保真导入展示）

### 1) PDF.js 多页 Viewer
- 路由：`/viewer/pdf/[file_id]`
- 实现方式：页面 iframe 内嵌官方 `pdfjs-dist` 的 `viewer.html`，并通过同源受控 URL 加载 PDF：
  - `viewer.html?file=<encodeURIComponent(/api/files/{id}/download?token=...)>`
- 目标：支持滚动浏览多页、翻页/缩放/页码等基本能力（不再只渲染第一页）。

### 2) Markdown(zip) 导入规范（md + 本地资源保真）
支持两种导入方式：
1. **单个 `.md`**：仅当 Markdown 内 **没有任何相对路径资源引用** 时允许导入；否则返回 400，并提示上传 zip。
2. **`.zip`（推荐）**：包含 1 个 Markdown 主文件 + 图片/资源目录（保持相对路径）。导入时将：
   - 选择主 Markdown：
     - 优先 `main.md`；否则 zip 内只有 1 个 `.md` 则使用；若多个则报错并返回候选列表
   - 解析并收集本地资源引用（至少覆盖）：
     - 行内图片 `![](./images/a.png)`
     - reference-style 图片 `![alt][id]` + `[id]: ./images/a.png`
     - HTML `<img src="./images/a.png">`
   - 对所有相对路径资源：从 zip 解压目录读取对应文件，上传到 MinIO，生成受控访问 URL
   - 重写 Markdown 中的路径为受控 URL（保证页面渲染时图片可显示）
   - 若发现引用了 zip 内不存在的本地资源：导入失败并返回缺失列表，避免悄悄丢图
   - 远程图片（http/https/data:）保持原链接不动

### 3) TeX(zip) 主文件自动识别
- 若 zip 根目录存在 `main.tex`：优先使用（兼容旧逻辑）
- 若不存在 `main.tex`：
  - zip 内仅 1 个 `.tex`：使用它
  - 多个 `.tex`：选择同时包含 `\documentclass` 与 `\begin{document}` 的那个
  - 若仍不唯一：导入失败，返回候选 tex 列表，并允许通过表单字段 `main_tex=paper.tex` 指定主文件
